\doxysubsection{Detailed Description}
Setting and querying serial port parameters.

See \doxylink{port_config_8c-example}{port\+\_\+config.\+c} for a working example of port configuration.

You should always configure all settings before using a port. There are no default settings applied by libserialport. When you open a port it may have default settings from the OS or driver, or the settings left over by the last program to use it.

You should always set baud rate, data bits, parity and stop bits.

You should normally also set one of the preset sp\+\_\+flowcontrol flow control modes, which will set up the RTS, CTS, DTR and DSR pin behaviours and enable or disable XON/\+XOFF. If you need an unusual configuration not covered by the preset flow control modes, you will need to configure these settings individually, and avoid calling \doxylink{group___configuration_gad055bc163037becad06b32fcad7f4c84}{sp\+\_\+set\+\_\+flowcontrol()} or \doxylink{group___configuration_ga7bbb731037e2981ef9859f1ebd7978e3}{sp\+\_\+set\+\_\+config\+\_\+flowcontrol()} which will overwrite these settings.

A port must be opened before you can change its settings.

There are two ways of accessing port settings\+:\hypertarget{group___configuration_autotoc_md26}{}\doxysubsubsection{\texorpdfstring{Configuration structures}{Configuration structures}}\label{group___configuration_autotoc_md26}
You can read and write a whole configuration (all settings at once) using \doxylink{group___configuration_gaa508d4258f8bd5b0c79c8c87d28ec3f5}{sp\+\_\+get\+\_\+config()} and \doxylink{group___configuration_ga386651d3bc8c4e2821973d4e021711ac}{sp\+\_\+set\+\_\+config()}. This is handy if you want to change between some preset combinations, or save and restore an existing configuration. It also ensures the changes are made together, via an efficient set of calls into the OS -\/ in some cases a single system call can be used.

Use accessor functions like \doxylink{group___configuration_gade751d4337c345c7a0408857774478bc}{sp\+\_\+get\+\_\+config\+\_\+baudrate()} and \doxylink{group___configuration_ga003ec5a37149263feb444b9e4cd6c259}{sp\+\_\+set\+\_\+config\+\_\+baudrate()} to get and set individual settings from a configuration.

For each setting in a port configuration, a special value of -\/1 can be used, which will cause that setting to be left alone when the configuration is applied by \doxylink{group___configuration_ga386651d3bc8c4e2821973d4e021711ac}{sp\+\_\+set\+\_\+config()}.

This value is also be used by \doxylink{group___configuration_gaa508d4258f8bd5b0c79c8c87d28ec3f5}{sp\+\_\+get\+\_\+config()} for any settings which are unconfigured at the OS level, or in a state that is not representable within the libserialport API.

Configurations are allocated using \doxylink{group___configuration_gaf294f646810d9dc8b29b3d39b778afa0}{sp\+\_\+new\+\_\+config()} and freed with \doxylink{group___configuration_ga2b48ea0772650666b9d2346c319d88a2}{sp\+\_\+free\+\_\+config()}. You need to manage them yourself. When a new configuration is allocated by \doxylink{group___configuration_gaf294f646810d9dc8b29b3d39b778afa0}{sp\+\_\+new\+\_\+config()}, all of its settings are initially set to the special -\/1 value.\hypertarget{group___configuration_autotoc_md27}{}\doxysubsubsection{\texorpdfstring{Direct functions for changing port settings}{Direct functions for changing port settings}}\label{group___configuration_autotoc_md27}
As a shortcut, you can set individual settings on a port directly by calling functions like \doxylink{group___configuration_ga58ede476a227c8d4db98020da34bb8d0}{sp\+\_\+set\+\_\+baudrate()} and \doxylink{group___configuration_ga82a73f4dd935c3ccb314c1a83157e33c}{sp\+\_\+set\+\_\+parity()}. This saves you the work of allocating a temporary config, setting it up, applying it to a port and then freeing it.
